# 消融实验执行方案

> 最后更新：2026-04-15
> 适用范围：当前毕业论文主线实验
> 主实验对象：`morph` 数据集，`UNI + CLAM_SB`

## 1. 文档目的

本方案用于正式规定当前论文阶段需要完成的消融实验，包括：

- 做哪些实验
- 每个实验为什么必须做
- 对应训练命令
- 对应评估命令
- 结果目录应如何命名

本方案默认以当前最成熟结果线 `morph_uni_clam_sb_v3` 为参照，不再把 `nanchang` 或 `all` 作为第一优先级消融对象。

## 2. 命名规范

从本文件开始，后续新增文档、实验计划、结果整理文件统一采用中文命名；但训练命令中的 `--exp_code` 保持英文/ASCII 命名。

### 2.1 文档命名

建议格式：

- `论文提纲.md`
- `实验记录总表.md`
- `消融实验执行方案.md`
- `结果整理（2026-04-15）.md`

### 2.2 实验命名

训练命令中的 `--exp_code` 统一采用英文/ASCII，建议采用：

- `morph_baseline_v3`
- `morph_mil_v3`
- `morph_no_inst_v3`
- `morph_no_pca_v3`
- `morph_no_aug_v3`
- `morph_bag_weight_0_5_v3`
- `morph_no_warmup_v3`

说明：

- 文档描述、表格标题、实验说明全部可以使用中文
- `exp_code` 保持英文更稳妥，便于脚本兼容、批量统计和后续复现

## 3. 基准实验

所有消融实验都应以同一个参考实验为基线，只改变一个因素。

### 3.1 基线实验名称

- 文档显示名：`形态基线 v3`
- `exp_code`：`morph_baseline_v3`

### 3.2 基线训练命令

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --task task_3_dlbcl_coo \
  --dataset morph \
  --feature_type uni \
  --data_root_dir features \
  --results_dir results \
  --exp_code morph_baseline_v3 \
  --model_type clam_sb \
  --seed 1 \
  --k 10 \
  --lr 5e-5 \
  --drop_out 0.5 \
  --reg 1e-3 \
  --bag_loss ce \
  --monitor_metric val_auc \
  --use_pca \
  --pca_dim 256 \
  --feature_noise_std 0.02 \
  --feature_dropout 0.1 \
  --patch_keep_ratio 0.8 \
  --max_patches_per_bag 512 \
  --warmup_bag_only_epochs 10 \
  --attention_entropy_weight 0.001 \
  --label_smoothing 0.05 \
  --bag_weight 0.7 \
  --B 8
```

### 3.3 基线实验作用

该实验是所有消融实验的对照组，用于回答：

- 当前最优设置到底是什么
- 去掉某个模块后效果相对下降多少
- 论文中的主结果表和消融表应以哪个配置为参照

## 4.1 实验一：MIL 对照实验

### 实验名称

- 文档显示名：`形态 MIL 对照 v3`
- `exp_code`：`morph_mil_v3`

- CLAM 是否真的优于标准 MIL

如果这个实验缺失，论文很难证明 CLAM 本身有必要保留。

```bash
  --drop_out 0.5 \
  --reg 1e-3 \
```

## 4.4 实验四：去掉特征增强

- 文档显示名：`形态去增强 v3`
- `exp_code`：`morph_no_aug_v3`

### 为什么要做

该实验用于回答：

- 当前性能提升是否依赖特征增强与 bag 扰动
- 特征级正则化是否真正缓解了过拟合

这是验证“训练策略是否有效”的核心实验。

### 重要说明

由于当前 [main.py](../main.py) 对 DLBCL 任务设置了默认增强参数，以下参数不能简单写成 `0` 或 `1.0`，否则会被默认值覆盖。因此本实验使用“不会触发默认覆盖”的写法：

- `--feature_noise_std -1`
- `--feature_dropout -1`
- `--patch_keep_ratio 1.01`
- `--max_patches_per_bag 100000`

### 训练命令

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --task task_3_dlbcl_coo \

  --lr 5e-5 \
  --drop_out 0.5 \
  --reg 1e-3 \
  --bag_loss ce \
```

## 6. 不建议当前优先做的实验

以下实验暂不列为当前论文阶段优先任务：

- SWA 消融
- model size 消融
- `all` 数据集上的完整消融

原因：

- 当前论文主线应先围绕最成熟的 `morph` 结果收口
- 如果主线证据链还不完整，继续扩展支线会显著分散时间

## 7. 建议执行顺序

建议按以下顺序推进：

1. `morph_mil_v3`
2. `morph_no_inst_v3`
3. `morph_no_pca_v3`
4. `morph_no_aug_v3`
5. `morph_bag_weight_0_5_v3`
6. `morph_no_warmup_v3`（时间允许再做）

如果时间充足，可再补：

7. `morph_bag_weight_0_9_v3`

## 10. 当前结论

当前阶段最合适的做法不是继续扩张实验范围，而是：

- 围绕 `morph` 这条最成熟主线完成一套完整消融
- 让论文能够明确说明“提升来自哪里”
- 把主结果、消融结果、解释性结果组织成一条闭合证据链

本文件后续将作为当前论文消融实验的正式执行标准。

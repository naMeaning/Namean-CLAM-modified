# DLBCL 训练过拟合专项优化方案

## 1. 文档目的

本文档专门回答一个问题：

当前项目中，为什么会出现“训练集准确率 99% 甚至接近 100%，但验证集/测试集效果很差”，以及应该如何通过数据增强、动态 PCA 降维、bag 级增强和训练约束来解决。

这份文档只聚焦抗过拟合方案，不展开讨论交叉验证设计、source-aware split 和 checkpoint 选模问题。那些内容已经放在另一份总分析文档里。

---

## 2. 先说结论

这个项目里，训练阶段直接读取的是 patch 特征 `.pt` 文件，而不是原始图像 patch。

因此：

1. 不能把普通图像分类那套增强方法直接当作主解法。
2. 当前最有效的抗过拟合路线，应该优先放在：
   - 特征级增强
   - bag 级增强
   - 动态 PCA 降维
   - 分阶段训练
   - 一致性约束

一句话概括：

当前项目要解决“训练集几乎完美、测试集很差”，重点不是继续提高模型拟合能力，而是主动削弱模型记忆训练集噪声的能力。

---

## 3. 为什么这个项目的数据增强不能只讲图像增强

当前项目的训练数据流是：

1. 先从 WSI patch 提取特征。
2. 训练阶段在 `dataset_modules/dataset_generic.py` 中直接读取 `pt_files/<slide_id>.pt`。
3. 每个 slide 对应一个 bag，bag 内是若干 patch embedding。
4. CLAM/MIL 训练时直接处理这些 embedding。

也就是说，训练时模型真正看到的是：

```text
N x D 的 patch feature 矩阵
```

其中：

- `N` 是一个 slide 中 patch 的数量
- `D` 通常是 `1024`

因此：

1. 训练代码阶段已经看不到像素。
2. 训练阶段不能直接做旋转、翻转、颜色抖动。
3. 如果不重做特征提取，最有效的增强应该直接作用在：
   - patch 特征本身
   - patch 子集
   - bag 表征

这也是为什么“要不要做数据增强”在这个项目里，真正应该理解成：

“要不要做特征空间增强和 bag 空间增强”

而不是传统图像增强。

---

## 4. 过拟合的本质是什么

当前出现“训练集 99%，测试集很差”，本质上说明：

1. 模型有足够能力把训练集中的判别模式记住。
2. 这些模式里有一部分并不是稳定可泛化的病理信号。
3. 模型过度依赖了训练集中特定 patch、特定特征方向、特定 bag 结构。

在 CLAM 里，这种过拟合尤其容易表现为：

1. attention 过早聚焦到少数高响应 patch。
2. instance clustering 强化了少数“捷径 patch”。
3. bag-level 分类器过度依赖高维 embedding 中的少数噪声方向。

因此，真正的解决思路应该围绕下面三点：

1. 让输入表征更难被死记硬背。
2. 让同一个 bag 在训练过程中呈现多个不同视图。
3. 约束模型不要过度依赖极少数 patch 或极少数特征维度。

---

## 5. 第一层方案：特征级数据增强

这一层是最适合当前项目优先落地的方案，因为它最贴合现有代码结构。

## 5.1 Feature Noise

### 原理

对每个 patch feature 加小幅高斯噪声：

```text
x' = x + eps, eps ~ N(0, sigma^2)
```

### 推荐参数

- `sigma = 0.01 ~ 0.05`

### 作用

1. 防止模型记住某个 patch embedding 的精确数值。
2. 提高模型对 feature 浮动的鲁棒性。
3. 在不改变语义标签的情况下，扩充训练视图。

### 注意事项

1. 只能在训练集启用。
2. 噪声不能过大，否则会直接破坏特征语义。
3. 推荐先从 `0.02` 开始。

---

## 5.2 Feature Dropout

### 原理

对每个 patch embedding 的一部分维度随机置零。

例如：

- 每个 patch 随机丢弃 `10%` 的特征维度

### 作用

1. 防止模型过度依赖个别 embedding 维度。
2. 相当于输入侧的 dropout。
3. 比只在网络内部加 dropout 更直接地约束输入表示。

### 推荐范围

- `drop_prob = 0.05 ~ 0.2`

### 适用性

这个方法对当前项目非常适合，因为当前特征维度高、病例数少，高维输入本身就是过拟合来源之一。

---

## 5.3 Patch Dropout

### 原理

训练时对一个 bag 中的 patch 随机丢弃一部分。

例如：

- 每次只保留 `70% ~ 90%` 的 patch

### 作用

1. 防止模型把训练集中固定的几个高注意力 patch 当成唯一依据。
2. 提高模型对 patch 缺失的鲁棒性。
3. 让同一张切片在不同 epoch 呈现不同 bag 结构。

### 为什么它特别重要

MIL/CLAM 过拟合时，经常不是“整袋都记住”，而是“只记住几个特别有区分度但未必稳健的 patch”。

Patch dropout 正是针对这个问题。

---

## 5.4 Patch Subsample

### 原理

不是每次都把一个 slide 的所有 patch 全部喂给模型，而是在训练时只采样其中一个子集。

例如：

- 每个 bag 最多保留 `256` 或 `512` 个 patch
- 或只保留一定比例

### 作用

1. 让同一个 bag 在不同迭代中形成多个视图。
2. 降低模型对某一固定 patch 组合的记忆。
3. 还可以降低显存和训练开销。

### 推荐方式

对于 patch 数很多的 slide，这种方法通常应该默认开启。

对当前项目而言，这往往比简单再把 dropout 从 `0.25` 调到 `0.5` 更有效。

---

## 6. 第二层方案：bag 级增强

特征级增强针对单个 patch embedding，而 bag 级增强是对整个 slide 的 patch 集合结构做扰动。

## 6.1 Bag Mixup

### 原理

从同一类别中取两个 bag，随机抽取各自一部分 patch，拼成一个新的 bag，标签保持不变。

### 为什么建议先做“同类 mixup”

因为：

1. 跨类 mixup 在 MIL 任务中容易引入语义冲突。
2. 同类 mixup 更稳，不容易破坏标签一致性。

### 作用

1. 扩展同类 bag 的内部结构多样性。
2. 防止模型只记住某个特定病例的 patch 组合模式。

### 适用场景

更适合在病例数较少但单类内部形态差异较大的任务上使用。

---

## 6.2 Hard Patch Drop

### 原理

训练若干 epoch 后，利用模型当前的 attention 分数，把注意力最高的一部分 patch 中随机屏蔽若干个，再做训练。

### 作用

1. 打破模型对极少数高注意力 patch 的依赖。
2. 强迫模型学习更分散、更稳定的证据。
3. 对 CLAM 特别有针对性。

### 适用阶段

不建议从 epoch 0 就启用。

更推荐：

- 先正常训练若干 epoch
- 再在中后期逐步加入

---

## 6.3 Two-View Bag Consistency

### 原理

对同一个 bag 生成两个不同增强视图：

- 视图 A：feature noise + patch dropout
- 视图 B：feature dropout + patch subsample

然后要求模型对两个视图的输出尽量一致。

### 可选损失

- KL divergence
- JS divergence
- logits/probability MSE

### 作用

1. 强迫模型学习稳定判别模式。
2. 降低对随机局部细节的依赖。
3. 是当前项目里非常值得加的正则项。

### 为什么适合这个项目

因为当前一个 slide 本身就是一个 bag，很自然可以做“同一 bag 的不同视图一致性”。

---

## 7. 第三层方案：动态 PCA 降维

你提到“动态 PAC 降维”，结合当前项目最合理、最可实现的方案，应该落实为“动态 PCA 降维”。

如果你说的不是 PCA，而是别的 PAC 方法，那么需要单独重新定义；但在当前仓库中，PCA 是最合适的可落地选择。

## 7.1 为什么要做 PCA

当前输入维度大多是：

- `1024`

而病例数并不大，尤其：

- `nanchang` 只有 `50` 个 case

这种情况下，高维 embedding 很容易带来两个问题：

1. 低方差维度里混入噪声。
2. 分类器过度利用训练集特有方向。

PCA 的价值在于：

1. 去掉低方差噪声维度。
2. 降低输入自由度。
3. 削弱模型的记忆能力。

---

## 7.2 PCA 必须怎么做才正确

PCA 必须按 fold 独立拟合，并且只能使用训练集。

正确流程是：

1. 当前 fold 先划分出 train/val/test。
2. 收集 train split 中所有 patch 特征。
3. 仅用 train patch 拟合 PCA。
4. 用这个 PCA 同时变换 train、val、test。

绝对不能：

1. 用全数据拟合 PCA。
2. 让 val/test 参与 PCA 拟合。

否则会产生信息泄漏。

---

## 7.3 推荐 PCA 维度

建议按数据集规模设置：

| 数据集 | 推荐 PCA 维度 |
|---|---:|
| `nanchang` | 128 或 256 |
| `morph` | 256 或 384 |
| `all` | 384 或 512 |

建议从更保守的维度开始试。

对于 `nanchang`，首选：

- `256`

---

## 7.4 什么叫动态 PCA

在本项目里，推荐的“动态 PCA”不是每个 epoch 重新拟合 PCA，而是：

1. 每个 fold 只拟合一次 PCA 基。
2. 训练过程中逐步增加保留维度。

例如：

- epoch 0-9：保留前 `128` 维
- epoch 10-19：保留前 `256` 维
- epoch 20+：保留前 `384` 维

### 这样做的好处

1. 训练早期先只看最稳定的主成分。
2. 模型先学粗粒度判别边界。
3. 后期再逐步释放更多细节信息。

这比一开始就直接给足 1024 维更不容易过拟合。

---

## 7.5 是否建议 whitening

建议作为可选项，不建议默认开启。

原因：

1. whitening 会改变各主成分尺度。
2. 小样本场景下不一定稳定。
3. 先验证普通 PCA 是否有效，再考虑 whitening。

推荐：

- 默认 `whiten=False`

---

## 8. 第四层方案：训练策略约束

数据增强和降维只是第一步，训练过程本身也要有抗过拟合约束。

## 8.1 Warmup Bag-Only

### 原理

训练前若干 epoch，只用 bag-level loss，不启用 instance clustering。

之后再恢复标准 CLAM。

### 作用

1. 先让模型学会 slide-level 判别。
2. 避免 instance-level 信号过早把噪声强化。
3. 对小数据集更稳。

### 推荐

- `warmup_bag_only_epochs = 10`

对 `nanchang` 尤其值得尝试。

---

## 8.2 降低 `k_sample`

当前 CLAM 使用 top-k patch 进行 instance 训练。

如果 `k_sample` 太大，模型可能会过分依赖少数高注意力 patch。

建议在小数据集上尝试：

- `8 -> 4`
- 或 `8 -> 6`

这样会让 instance clustering 更保守。

---

## 8.3 Label Smoothing

### 原理

对 bag-level CE 损失引入轻微 label smoothing。

### 推荐范围

- `0.05 ~ 0.1`

### 作用

1. 降低输出过度自信。
2. 减缓训练精度过快冲到 100%。
3. 对小验证集任务比较有帮助。

---

## 8.4 Attention Entropy Regularization

### 原理

给 attention 分布加一个熵约束，防止注意力过于尖锐。

### 作用

1. 防止模型只盯住极少数 patch。
2. 让注意力分布更平滑。
3. 同时改善可解释性稳定性。

### 注意

这个正则不宜过强，否则可能破坏真正有意义的聚焦行为。

建议小权重开始：

- `1e-4 ~ 1e-3`

---

## 8.5 Consistency Loss

对于同一 bag 的两个增强视图，加一个一致性损失：

```text
L_total = L_bag + lambda_inst * L_inst + lambda_cons * L_cons
```

其中：

- `L_bag` 是 slide-level 分类损失
- `L_inst` 是 instance clustering 损失
- `L_cons` 是两视图一致性损失

推荐：

- `lambda_cons = 0.1 ~ 0.3`

作用：

1. 约束模型学习对局部扰动不敏感的表示。
2. 比单纯堆更复杂结构更有针对性。

---

## 9. 针对本项目的代码改造思路

下面把方案落到文件级。

## 9.1 建议新增文件

建议新增两个工具文件：

1. `utils/feature_aug.py`
2. `utils/pca_utils.py`

### `utils/feature_aug.py` 建议包含

- `add_gaussian_noise(features, std)`
- `feature_dropout(features, drop_prob)`
- `patch_dropout(features, keep_ratio)`
- `random_subsample_patches(features, max_patches or keep_ratio)`
- `make_two_views(features, config)`

### `utils/pca_utils.py` 建议包含

- `fit_pca_from_train_split(train_dataset, n_components, whiten=False)`
- `apply_pca(features, pca_model, n_components=None)`
- `get_epoch_pca_dim(epoch, schedule)`

---

## 9.2 `main.py` 应增加的参数

建议增加：

```bash
--feature_noise_std
--feature_dropout
--patch_keep_ratio
--max_patches_per_bag
--use_bag_consistency
--consistency_weight
--use_pca
--pca_dim
--pca_whiten
--pca_schedule
--warmup_bag_only_epochs
--attention_entropy_weight
--label_smoothing
```

推荐起始值：

- `feature_noise_std = 0.02`
- `feature_dropout = 0.1`
- `patch_keep_ratio = 0.8`
- `max_patches_per_bag = 512`
- `use_bag_consistency = True`
- `consistency_weight = 0.2`
- `use_pca = True`
- `pca_dim = 256` for `nanchang`
- `pca_whiten = False`
- `warmup_bag_only_epochs = 10`
- `attention_entropy_weight = 1e-3`
- `label_smoothing = 0.05`

---

## 9.3 `dataset_modules/dataset_generic.py` 应怎么改

当前 `Generic_MIL_Dataset.__getitem__()` 是整个增强入口的最佳落点之一。

建议修改思路：

1. 在数据集类中增加：
   - `feature_transform`
   - `pca_transform`
   - `training`
2. 如果是训练集：
   - 先应用 PCA
   - 再应用特征增强
3. 如果是验证/测试集：
   - 只应用 PCA
   - 不做随机增强

这样做的好处：

1. 所有输入侧扰动都集中在数据入口。
2. 验证/测试逻辑天然干净。
3. 后续维护简单。

---

## 9.4 `utils/utils.py` 应怎么改

主要涉及：

- `collate_MIL()`
- `get_split_loader()`

建议：

1. 训练时默认关闭 `weighted_sample`。
2. 支持 patch 子采样和 patch dropout。
3. 如果要做双视图一致性，可以额外实现：
   - `collate_MIL_two_views()`

输出形式为：

- `view1_features`
- `view2_features`
- `label`

这样训练循环就可以直接计算一致性损失。

---

## 9.5 `utils/core_utils.py` 应怎么改

这个文件需要承担训练约束逻辑。

建议新增三类能力：

### 1. 两视图一致性训练

支持同一 bag 两个增强视图前向，并计算 consistency loss。

### 2. Warmup Bag-Only

训练前若干 epoch：

- 只用 bag loss
- 暂不使用 instance clustering

之后再恢复标准 CLAM。

### 3. Attention 熵正则

在 CLAM 训练过程中加入轻量 attention entropy penalty。

---

## 9.6 `models/model_clam.py` 应怎么改

模型本体改动应该放在后面做，不是第一优先级。

更推荐的改动方向是：

1. 适当保留或增强输入投影层的 dropout。
2. 允许 attention temperature 调节，减缓 attention 过快变尖。
3. 降低 `k_sample` 默认值。

但优先级低于：

1. 输入侧特征增强
2. 动态 PCA
3. 训练流程约束

---

## 10. 推荐的实施顺序

不要一开始就把所有技巧全部堆上去。

建议分阶段做。

## 第一阶段：最低风险、高收益

1. 关闭 `weighted_sample`
2. 加 `feature noise`
3. 加 `feature dropout`
4. 加 `patch dropout`
5. 加 `max_patches_per_bag`

目标：

先削弱最明显的训练集记忆化。

## 第二阶段：结构性抗过拟合

6. 加 PCA
7. 做动态维度调度
8. 加 warmup bag-only
9. 加 bag consistency

目标：

进一步提升泛化，不再让模型只依赖少数固定证据。

## 第三阶段：CLAM 专项优化

10. attention entropy regularization
11. 调小 `k_sample`
12. 比较：
   - `MIL`
   - `bag-only CLAM`
   - `标准 CLAM`

目标：

判断 instance clustering 是增益还是干扰。

---

## 11. 最值得优先跑的三组实验

如果只能先跑三组，建议如下。

## 方案 A：最稳 bag-only baseline

- `nanchang`
- `UNI`
- `CLAM_SB bag-only`
- `feature noise + patch dropout + PCA(256)`

意义：

先确认纯 bag-level 在强约束下能否稳住泛化。

## 方案 B：标准 CLAM 温和版

- `nanchang`
- `UNI`
- `标准 CLAM_SB`
- `warmup bag-only 10 epochs`
- `feature noise + patch dropout + PCA(256) + consistency loss`

意义：

确认 instance clustering 在更稳训练流程下是否真的有帮助。

## 方案 C：更大单域验证

- `morph`
- `UNI`
- `标准 CLAM_SB`
- `feature noise + patch dropout + PCA(384) + consistency loss`

意义：

用更大的单域数据验证方法是否有效，而不是只在 `nanchang` 的 50 case 上观察波动。

---

## 12. 最终建议

对于当前这个项目，如果你只想抓住最核心的一点，那就是：

不要把“数据增强”理解成重新去做大量图像旋转翻转，而应该把重点放在：

1. 特征级增强
2. bag 级增强
3. 动态 PCA 降维
4. warmup bag-only
5. 一致性约束

这套方案的目标不是把训练集精度做得更高，而是主动限制模型对训练集噪声的记忆能力。

对你当前这种“训练 99%，测试很差”的情况，这才是更符合项目结构、也更可能真正改善结果的方向。

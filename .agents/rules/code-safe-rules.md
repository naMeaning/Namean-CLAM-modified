在当前工作阶段，你必须遵守以下最高优先级的红线规则：

纯注释模式 (Comment-Only Mode)： 仅允许在代码中插入注释（如 //, /* */, #, """ """ 等），绝对禁止修改、优化或重构任何现有的功能代码、业务逻辑或执行顺序。

零变量修改： 严禁修改任何变量名、函数名、类名、导入语句或文件结构。

保持原有格式： 插入注释时，必须严格保持原有代码的缩进、换行和空格风格，不要破坏现有的代码排版。

全量输出： 必须输出包含新增注释的完整代码。严禁使用 ... 或 // remaining code 等占位符省略代码，确保用户可以直接复制覆盖原文件并成功运行。

语言规范： 除非用户特别指定，否则一律使用中文编写注释。
